## Codebase Patterns
- Feature flags use `z.enum(["true", "false"]).optional().default("false")` pattern in `env.mjs` with matching `runtimeEnv` entry
- Client components access env vars via `process.env.NEXT_PUBLIC_*` directly (Next.js inlines at build time)
- Navigation tabs are defined in `header.tsx` as a const array, active tab state lives in `atoms/navigation.ts`
- `turbo.json` `globalEnv` must include any new `NEXT_PUBLIC_*` vars for cache invalidation
- app.mento.org has no `check-types` script; run `tsc --noEmit` from its directory or `pnpm check-types` from root (covers packages only)
- `.env.local` is gitignored; `.env.example` is committed
- SDK version is managed via pnpm workspace catalog in `pnpm-workspace.yaml`; `packages/web3` uses `"catalog:"` reference; `apps/app.mento.org` has a direct version override
- SDK upgrades may introduce breaking type changes in existing hooks â€” always run `pnpm check-types` after upgrading
- Liquidity quote/transaction hooks use `deadline: bigint` (Unix timestamp) â€” transactions use `block.timestamp + BigInt(20 * 60)`, quotes use `Date.now()` based timestamps
- Features barrel is at `packages/web3/src/features/index.ts` â€” add new features alphabetically
- Swap uses `hooks/index.ts` barrel pattern; pools lists hooks individually in its `index.ts` â€” swap pattern is cleaner for borrow
- `BorrowService` constructor: `new BorrowService(publicClient: PublicClient, chainId: number)`
- Existing SDK factory is in `features/sdk.ts` with a simple cache by chainId; borrow sdk.ts follows same caching pattern
- Borrow component directory: `apps/app.mento.org/app/components/borrow/` â€” follows pools pattern with a top-level view component
- Borrow navigation atom at `components/borrow/atoms/borrow-navigation.ts` â€” uses `BorrowView` type from `@repo/web3`
- Imperative tx utils use `wagmi/actions` (`sendTransaction`, `estimateGas`, `waitForTransactionReceipt`); `Config` type from `"wagmi"`
- SDK `CallParams` strings must be cast: `to` â†’ `Address`, `data` â†’ `Hex`, `value` â†’ `BigInt(value || 0)`
- Leverage math functions are pure (no contract calls); on-chain data (price, CR, trove data) must be fetched separately and passed in
- All leverage bigint values use 18-decimal fixed-point (1e18 = 1.0 or 100%); convert JS number leverage factors via `Math.round(factor * 1000) * 1e18 / 1000`
- `useBorrowService()` returns `BorrowService | null` â€” always guard with `enabled: !!sdk` in useQuery hooks
- Formatting utils in `features/borrow/format.ts`: `formatDebtAmount`, `formatCollateralAmount`, `formatPrice`, `formatInterestRate`, `formatLtv` â€” all accept `bigint | null | undefined`
- StabilityPool ABI at `features/borrow/stability-pool/abi.ts` â€” exported as `stabilityPoolAbi` (camelCase)
- `DEBT_TOKEN_CONFIGS` in `features/borrow/types.ts` â€” keyed by symbol string, currently has GBPm only; typed as `Record<string, DebtTokenConfig>` so use `!` assertion when accessing by key
- Borrow shared UI components live in `apps/app.mento.org/app/components/borrow/shared/`
- @repo/ui `Select` pattern: `Select` > `SelectTrigger` > `SelectValue` + `SelectContent` > `SelectItem` (Radix-based)
- @repo/ui `Badge` variants: `default`, `secondary`, `destructive`, `outline`
- SDK borrow types: `BorrowPosition`, `LoanDetails`, `SystemParams`, `CallParams`, `OpenTroveParams`, `AdjustTroveParams`, `InterestRateBracket`, `TroveStatus`, `RiskLevel`
- Swap hooks use `useDebounce(amount, 350)` for input-dependent queries â€” follow same pattern for fee prediction hooks
- Bigint values in React Query keys must be `.toString()` converted â€” bigints aren't JSON-serializable
- Shared `useDebounce` utility at `packages/web3/src/utils/debounce.ts` â€” reuse it, don't create per-feature copies
- `pools-view.tsx` is the best pattern reference for new tab views â€” uses `max-w-5xl` container with top-decorations
- SDK pure math functions (getLoanDetails, calculateDebtSuggestions, getRedemptionRisk, etc.) are NOT top-level exports â€” import from `@mento-protocol/mento-sdk/dist/services/borrow/borrowMath`
- Derived hooks use `useMemo` (not useQuery) â€” they compute from existing query data, not fetch new data
- SDK borrow helpers (e.g., `resolveAddressesFromRegistry`) import from deep path: `@mento-protocol/mento-sdk/dist/services/borrow/borrowHelpers`
- For contract reads NOT covered by BorrowService, use wagmi `useReadContract`/`useReadContracts` with the ABI directly
- StabilityPool address resolved via `useStabilityPoolAddress` internal hook â€” `getBorrowRegistry` + `resolveAddressesFromRegistry`, cached with `staleTime: Infinity`
- Dashboard components live in `apps/app.mento.org/app/components/borrow/dashboard/` â€” card-level components for positions and deposits
- @repo/ui Card composable pattern: `Card` > `CardHeader` > `CardTitle` + `CardAction`, `CardContent` for body
- `borrow-view.tsx` is the top-level router for the borrow tab â€” reads `borrowViewAtom` and renders sub-views; `DebtTokenSelector` lives in the header
- `BorrowView` union discrimination: use `typeof view === "object"` for the `manage-trove` variant, string comparison for the rest
