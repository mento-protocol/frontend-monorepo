## Codebase Patterns
- Feature flags use `z.enum(["true", "false"]).optional().default("false")` pattern in `env.mjs` with matching `runtimeEnv` entry
- app.mento.org has no `check-types` script; run `tsc --noEmit` from its directory or `pnpm check-types` from root (covers packages only)
- SDK version `^3.0.0-beta.18`; managed via pnpm workspace catalog in `pnpm-workspace.yaml`
- Features barrel is at `packages/web3/src/features/index.ts` — add new features alphabetically
- `BorrowService` constructor: `new BorrowService(publicClient: PublicClient, chainId: number)`
- `useBorrowService()` returns `BorrowService | null` — always guard with `enabled: !!sdk` in useQuery hooks
- Borrow hooks barrel at `packages/web3/src/features/borrow/hooks/index.ts` — exports alphabetically
- Borrow component directory: `apps/app.mento.org/app/components/borrow/` — dashboard/, shared/, atoms/
- `borrow-view.tsx` is the top-level router for the borrow tab — reads `borrowViewAtom` and renders sub-views
- `BorrowView` union discrimination: use `typeof view === "object"` for the `manage-trove` variant, string comparison for the rest
- Borrow navigation atom at `components/borrow/atoms/borrow-navigation.ts` — uses `BorrowView` type from `@repo/web3`
- Imperative tx utils use `wagmi/actions` (`sendTransaction`, `estimateGas`, `waitForTransactionReceipt`); `Config` type from `"wagmi"`
- SDK `CallParams` strings must be cast: `to` → `Address`, `data` → `Hex`, `value` → `BigInt(value || 0)`
- `sendSdkTransaction` and `waitForTx` at `features/borrow/tx-flows/send-tx.ts` — already built in Phase 0
- SDK pure math functions: import from `@mento-protocol/mento-sdk/dist/services/borrow/borrowMath`
- SDK borrow helpers (e.g., `resolveAddressesFromRegistry`): import from `@mento-protocol/mento-sdk/dist/services/borrow/borrowHelpers`
- `getBorrowRegistry(chainId, symbol)` is exported from top-level `@mento-protocol/mento-sdk`
- StabilityPool ABI at `features/borrow/stability-pool/abi.ts` — exported as `stabilityPoolAbi` (camelCase)
- StabilityPool address resolved via `useStabilityPoolAddress` internal hook — `getBorrowRegistry` + `resolveAddressesFromRegistry`, cached with `staleTime: Infinity`
- Formatting utils in `features/borrow/format.ts`: `formatDebtAmount`, `formatCollateralAmount`, `formatPrice`, `formatInterestRate`, `formatLtv` — all accept `bigint | null | undefined`
- `DEBT_TOKEN_CONFIGS` in `features/borrow/types.ts` — typed as `Record<string, DebtTokenConfig>` so use `!` assertion when accessing by key
- Bigint values in React Query keys must be `.toString()` converted — bigints aren't JSON-serializable
- Shared `useDebounce` utility at `packages/web3/src/utils/debounce.ts` — reuse it, don't create per-feature copies
- @repo/ui `Select` pattern: `Select` > `SelectTrigger` > `SelectValue` + `SelectContent` > `SelectItem` (Radix-based)
- @repo/ui `Badge` variants: `default`, `secondary`, `destructive`, `outline`
- @repo/ui Card composable pattern: `Card` > `CardHeader` > `CardTitle` + `CardAction`, `CardContent` for body
- SDK borrow types: `BorrowPosition`, `LoanDetails`, `SystemParams`, `CallParams`, `OpenTroveParams`, `AdjustTroveParams`, `InterestRateBracket`, `TroveStatus`, `RiskLevel`
- Derived hooks use `useMemo` (not useQuery) — they compute from existing query data, not fetch new data
- For contract reads NOT covered by BorrowService, use wagmi `useReadContract`/`useReadContracts` with the ABI directly
- Swap write hooks pattern: `useMutation` + `useQueryClient()` for invalidation + toast on success — follow for borrow write hooks
- `atomWithStorage` from `jotai/utils` — use for localStorage-persisted atoms (e.g., flow state surviving page refresh)
- Write hook template: `useMutation` + `executeFlow(wagmiConfig, setFlowAtom, flowId, operation, account, stepDefs)` + `onSuccess` invalidates + toast
- Approval step pattern: `buildTx` checks allowance, returns `null` if sufficient (engine auto-skips), returns `sdk.buildCollateralApprovalParams(symbol, amount)` if not
- Debt token approval: `sdk.getDebtAllowance(symbol, owner, spender)` and `sdk.buildDebtApprovalParams(symbol, spender, amount)` — both require explicit spender (BorrowerOperations address)
- Resolve BorrowerOperations address imperatively: `getChainId(wagmiConfig)` + `getPublicClient(wagmiConfig)` from `wagmi/actions`, then `getBorrowRegistry` + `resolveAddressesFromRegistry`
- `borrowViewAtom` lives in app layer (`components/borrow/atoms/borrow-navigation.ts`), NOT in packages/web3 — hooks in web3 cannot import it; navigation must be handled by calling components

- SP tx builders use `encodeFunctionData` from viem with the local `stabilityPoolAbi` — no SDK dependency needed for SP writes
- App components use `@repo/web3` dist — rebuild web3 package (`pnpm --filter @repo/web3 build`) after adding new exports before running app tsc
- FlowStep component uses inline SVGs for status icons — no external icon library dependency (Tailwind `animate-spin` for spinner)
- Block explorer tx link pattern: `${useExplorerUrl()}/tx/${txHash}` — `useExplorerUrl` from `@repo/web3`
- `FlowDialog` is self-managing — just render `<FlowDialog />` in borrow-view, no props needed; visible when `borrowFlowAtom` is not null
- `@repo/ui` Dialog components: `Dialog`, `DialogContent`, `DialogHeader`, `DialogTitle`, `DialogDescription`, `DialogFooter` — Radix-based, use `open` prop for programmatic control
- `buildSpDeposit` encodes `provideToSP(uint256, bool)`, `buildSpWithdraw` encodes `withdrawFromSP(uint256, bool)`
- SP write hooks resolve SP address imperatively inside mutationFn using `getBorrowRegistry` + `resolveAddressesFromRegistry` — `addresses.stabilityPool`
- Debt token approval for SP deposit: spender is SP address (not BorrowerOperations) — `sdk.getDebtAllowance(symbol, owner, spAddress)` and `sdk.buildDebtApprovalParams(symbol, spAddress, maxUint256)`
